## 引言

\begin{figure}[t]
    \centering
    \includegraphics[width=0.955\linewidth]{figs/teaser_v12.pdf}
    \caption{
        \small
        **从像素到预测性心智。**
        我们超越了*仅限语言*的理解，构想了一种作为连续生活世界的一部分进行观察、记忆和推理的多模态智能。
        它始于*语义感知*：命名并描述所见之物。*流式事件认知*则更进一步，实现了对连续输入流的全天候感知，集成了记忆并支持主动响应。*空间认知*捕捉视频中隐含的 3D 结构，能够对物体、配置和度量进行推理。最后，*预测性世界模型*应运而生，它从经验中被动学习，通过预测和惊奇进行更新，并保留信息以供未来使用。
        **下方插图：**视频是理想的实验领域。模型必须从帧级的问答（Q\&A）进化到构建隐式世界模型，从而实现更深层的空间推理，扩展到无界视界，并获得足以媲美并最终超越人类视觉智能的超感能力（supersensing）。
    }\label{fig:teaser}
\end{figure}

视频不仅仅是孤立的帧序列。它是隐藏的、演化的 3D 世界在像素上的持续、高带宽投影~[gibson2014ecological, marr2010vision]。尽管通过将强大的图像编码器与语言模型相结合~[achiam2023gpt, team2024gemini, anthropic_claude35_sonnet_2024, liu2023visual, tong2024cambrian]，多模态大语言模型（MLLMs）取得了飞速发展，但大多数视频扩展模型~[wang2024internvideo2, li2024llava, bai2025qwen2]在根本上仍受到限制。它们仍将视频视为稀疏的帧，对空间结构和动力学的表示不足~[yang2024think]，且过度依赖文本召回~[zohar2024apollo]，从而忽略了使视频模态具有独特威力的核心要素。

 在本文中，我们认为迈向真正的多模态智能需要从以语言为中心的感知转向空间*超感能力（supersensing）：*这种能力不仅在于观察，还在于通过持续的感官经验构建、更新隐性 3D 世界模型并进行预测。我们并不声称在此实现了超感能力；相反，我们迈出了通往该目标的第一步，通过阐明可能引导至这一方向的发展路径，并展示该路径上的早期原型：

[start=0,leftmargin=1em,nosep]
    - **\color{gray** （仅限语言理解）：} 无感官能力；推理局限于文本和符号。当前的 MLLMs 已经超越了这一阶段，但仍保留着其偏见的痕迹。
    - **语义感知：** 将像素解析为物体、属性和关系。这对应于 MLLMs 中强大的多模态*“看图说话（show and tell）”*能力。
    - **流式事件认知：** 处理实时的、无界的流数据，同时主动解释并响应正在发生的事件。这与将 MLLMs 打造为实时助手的努力相一致。
    - **隐式 3D 空间认知：** 将视频理解为 3D 世界的投影。智能体必须知道存在什么、在何处、事物如何关联以及配置随时间如何变化。当今的视频模型在这方面仍然受限。
    - **预测性世界建模：** 大脑通过基于先验期望预测潜在的世界状态来进行*无意识推理*~[von1867handbuch]。当这些预测被违背时，惊奇（surprise）会引导注意力、记忆和学习~[friston2010free,stahl2015observing,kennedy2024prediction]。然而，当前的多模态系统缺乏一个能够预判未来状态，并利用惊奇来组织感知以进行记忆和决策的内部模型。

我们的论文分为三个部分。
**第一部分**（\S~\ref{sec:benchmark}），我们通过空间超感分级体系重新审视现有的基准测试。我们发现大多数基准测试对应于前几个阶段，而诸如 VSI-Bench~[yang2024think] 等基准开始探索空间推理。然而，没有一个基准能够充分应对预测性世界建模这一最终的关键阶段。
为了使这一差距具体化并推动方法的转变，我们引入了 \vsisuper{}（VSI 代表*视觉空间智能*），这是一个用于空间超感的两部分基准测试：\vsisuper{} Recall（\texttt{\vso}）针对长时程的空间观察和召回，而 \vsisuper{} Count（\texttt{\vsc}）测试在不断变化的视角和场景中的持续计数。
这些任务由任意长度的时空视频构建，刻意抵制了主流的多模态方案；它们要求感知是*选择性*且*结构化*的，而非不加区别地累积。
我们展示了即使是目前最优秀的、支持长上下文的商业模型，在 \vsisuper{} 上的表现也差强人意。

**第二部分**（\S~\ref{sec:limits}），我们研究空间超感是否仅仅是一个数据问题。我们策划了 *\vsidata{*}，这是一个专注于图像和视频的空间指令微调语料库，我们使用它训练了 *\cambrianS*，这是一系列具有空间对齐能力的视频 MLLMs。
在当前的范式下，精心的数据设计和训练使 \cambrianS{} 在 \textsc{VSI-Bench} 上达到了最先进的空间认知水平（绝对增益超过 $30\%$），同时不牺牲通用能力。
尽管如此，\cambrianS{} 在 \vsisuper{} 上依然表现不佳，这表明虽然规模化奠定了关键基础，但仅靠规模化不足以实现空间超感。

这激发了**第三部分**，也是最后一部分（\S~\ref{sec:predictive-sensing}），我们提出了*预测性感知*，作为迈向新范式的第一步。
我们展示了一个基于自监督下一潜空间帧预测（next-latent-frame prediction）构建的概念验证解决方案。
在此，我们利用模型的预测误差，即“惊奇度（surprise）”，来实现两个关键功能：
(1) 记忆管理，通过将资源分配给意外事件；
(2) 事件分割，将无界的流数据切分为有意义的块。
我们证明，这种方法虽然简单，但在我们的两个新任务上显著优于如 Gemini-2.5 等强大的长上下文基准模型。尽管这不是最终的解决方案，但这一结果提供了有力的证据，表明通往真正超感能力的路径需要模型不仅能够*观察*，还能积极地预测并从世界中学习。

我们的工作具有以下贡献：
（**1**）我们定义了空间超感的分级体系，并引入了 \vsisuper{}，这是一个揭示当前范式局限性的超感基准测试。
（**2**）我们开发了 \cambrianS，这是一个推高了空间认知极限的最先进模型。 \cambrianS 作为一个强大的新基准，通过在我们的新基准测试上划定当前方法的边界，为新范式铺平了道路。
（**3**）我们提出了预测性感知作为 MLLMs 一个有前景的新方向，展示了利用模型惊奇度进行长时程空间推理比被动的上下文扩展更有效。

## 评测空间超感知
\label{sec:benchmark}

### 解构现有视频基准测试
\label{sec:benchmark:deconstructing}

MLLM 的最新进展导致视频问答（Video-QA）基准测试激增。然而，一个关键问题仍然存在：*现有的视频基准测试在多大程度上真正考察了视觉感知能力，而非仅仅测试语言先验？* 我们的诊断测试通过改变视觉输入的丰富程度和文本线索的信息量，将模型对视觉感知与语言先验的依赖分离开来。仅凭文本输入（例如，描述或盲测 MLLM）即可解决的基准测试更偏向于考察语言理解。相比之下，只有通过多帧输入才能回答的基准测试问题才需要真正的视觉感知。
我们使用基于图像的多模态大语言模型 Cambrian-1~[tong2024cambrian] 进行评估，这使我们能够探测底层任务需求，而不会将其与特定视频架构和后训练方案的能力混淆。

 我们为 Cambrian-1~[tong2024cambrian] 模型建立了几种输入视频的实验条件：

    - **\multipleul{多个**帧}：模型处理从视频片段中均匀采样的 32 帧。这是文献中表示视频输入的标准方法~[li2024llava]。
    - **\singleul{单个**帧}：模型仅处理给定视频片段的中间帧。此条件测试对最少量、上下文核心视觉信息的依赖。
    - **帧\captionul{描述**}：模型接收与相同的 32 个均匀采样帧相对应的文本描述，而不是视频帧。
    此条件旨在揭示在*没有*底层感知关联的情况下，任务的可解决程度。
    我们使用 Gemini-2.0-Flash API 来重新生成视频帧的描述。

为了使这些条件下的性能具有参考意义，我们引入了另外两个基准：

    - **\blindul{盲**测}：
        模型仅使用任务的问题来尝试完成任务。*忽略所有视觉输入*，也不使用视觉描述。
        该基准衡量模型基于其既有知识、语言先验以及基准测试问题中任何潜在偏差的性能。 
    - **\chanceul{随机**准确率}：这代表了在特定任务格式（例如，多项选择题）下通过随机猜测可以达到的准确率，作为性能的底线。

 我们通过比较这些条件和基准之间的性能，对每个基准测试的特征进行了细粒度分析。我们专注于以下关键对比（\texttt{diff(A,B)} $=$ \texttt{A-B}）： 

    - \texttt{diff($\bf x$, \blindul{盲测})},\ $\bf x \in \big\{$\multipleul{多个}, \singleul{单个}, \captionul{描述}$\big\}$ 
        以量化不同输入模态相对于盲测基准提供的提升； 
    - \texttt{diff($\bf x$, \chanceul{随机})},\ $\bf x \in \big\{$\multipleul{多个}, \singleul{单个}, \captionul{描述}$\big\}$ 
        以衡量相对于随机水平的性能增益； 
        - \texttt{diff(\multipleul{多个}, \captionul{描述})}
            以了解当前主流实践与强大的仅语言基准之间的性能差距。

\cref{fig:benchmark_analysis} (a-c) 中展示的结果表明，Cambrian-1~[tong2024cambrian]——一个没有任何视频后训练的\singleul{基于图像的 MLLM}——在许多基准测试中可以获得合理的性能，在某些情况下超过\chanceul{随机水平}准确率 10-30\%（参见 \cref{fig:benchmark_analysis}-g,h）。这表明这些基准测试所针对的许多知识可以通过标准的单图指令微调流水线获取。
尽管如此，在两个现有的数据集 VSI-Bench~[yang2024think] 和 Tomato~[shangguan2024tomato] 上，模型的性能低于随机水平。对于 VSI-Bench，这主要是因为其空间理解问题需要真正的视频感知以及针对性的数据策划和训练。对于 Tomato，这种表现不佳是预料之中的：该基准测试需要理解高帧率视频中的细粒度细节，使得在时间上大幅降采样的单帧和 32 帧输入显得不足。

采用\captionul{文本描述}代替视觉输入也产生了显著的性能提升，在 EgoSchema~[mangalam2023egoschema]、VideoMME~[fu2025video]、LongVideoBench~[wu2024longvideobench]、VideoMMMU~[hu2025video]、Perception Test~[patraucean2023perception] 和 MVBench~[li2024mvbench] 等基准测试上超过\chanceul{随机准确率} 20\% 以上（\cref{fig:benchmark_analysis}-i）。在将基准测试性能与\blindul{盲测}结果进行对比时，可以得出类似的结论（\cref{fig:benchmark_analysis}-d,f）。这种性能暗示这些基准测试主要探测的是从视频内容的文本摘要中可以推断出的能力。
解释使用“\multipleul{多个帧}”与“\captionul{帧描述}”之间的性能差异（\cref{fig:benchmark_analysis}-j）时，显著的正差值（有利于\multipleul{多帧输入}）标志着基准测试对细微视觉感知的需求。相反，较小或为负的差值（更有利于“\captionul{帧描述}”）则表明其更偏向以语言为中心的性质。我们的分析将 VideoMMMU、EgoSchema、VideoMME、Perception Test 和 LongVideoBench 归入后一类，表明它们可能依赖于*语言理解*而非视觉线索。一个显著的例外是 VSC，它对于当前的 MLLM 来说极具挑战性，以至于三种输入条件产生的性能都接近于零，从而排除了它们之间任何有意义的比较。

\finding{}{\faBookmark~~现有基准测试绝大多数关注语言理解和语义感知，而忽视了超感知所需的更高级的空间和时间推理。}

我们希望强调基准测试中固有的挑战，以及创建一个单一的、包罗万象的基准测试来评估每种能力是不切实际的。例如，对语言先验的依赖不应仅仅被视为一个缺点，因为获取丰富的世界知识并进行有效的检索在许多场景中无疑是有益的。我们认为，**视频基准测试不应被视为测量单一、统一的“视频理解”概念。相反，它们的设计和评估应该基于它们旨在评估的特定能力。** 因此，上述分析旨在指导任务的开发，从而更有效地推动向*空间超感知*的进展，这将是本文其余部分的中心焦点。

### \vsisuper: 迈向多模态大语言模型空间超感知的基准测试
\label{sec:benchmark:vsi-super}
引用 \cref{fig:teaser}，空间超感知要求 MLLM 具备四项关键能力：*语义感知*、*流式事件认知*、*隐式 3D 空间认知*以及*预测性世界建模*。然而，正如我们在 \cref{fig:benchmark_analysis} 中的分析所指出的，现有的视频问答基准测试主要评估语言理解和语义感知方面，这些方面更偏向于反应性且由特定任务驱动~[fu2025video,mangalam2023egoschema,hu2025video]。虽然最近的研究已经开始通过持续感知、记忆架构和主动回答来解决流式事件认知问题~[chen2024videollm,qian2025dispider,niu2025ovo,wu2024streambench,song2024moviechat,zhang2024flash]，但这种能力通常是在测试时工程化实现的，而不是模型自身原生的技能。
此外，尽管空间推理偶尔作为现有基准测试的一个类别出现，但这些任务很少能达到真正的空间认知水平，且远未探测到定义超感知的世界建模能力（\cref{fig:benchmark}）。
虽然 VSI-Bench~[yang2024think] 在检验空间认知方面迈出了初步步伐，但其视频仍为短视频且场景单一，既没有形式化该问题，也没有评估世界预测性建模这一核心能力。

为了揭示当前 MLLM 与空间超感知之间的差距，我们推出了 \vsisuper，这是一个包含两个部分的持续空间感知基准测试。这些任务对人类来说是直观且通常很容易的，只需观看并记录发生的事情，但对于机器来说仍然具有惊人的挑战性。它们要求在无边界的空间视频中对视觉信息进行选择性过滤和结构化积累，以保持连贯的理解并回答问题。重要的是，它们能够抵抗暴力上下文扩展，从而暴露了对真正空间推理的需求。我们在下文详细介绍这两个组成部分。

\paragraph{\vsisuper Recall：长程空间观察与回忆。}
\vso 基准测试要求 MLLM 观察长程时空视频，并顺序回忆一个异常物体的位置。
如 \cref{fig:sor_benchmark_illustration} 所示，为了构建该基准测试，人类标注者使用图像编辑模型（即 Gemini~[comanici2025gemini]）在捕捉室内环境漫游的视频中~[dai2017scannet,yeshwanth2023scannet++,dehghan2021arkitscenes]，将令人惊讶或不合时宜的物体（例如泰迪熊）插入到四个不同的帧（和空间位置）中。然后将此编辑后的视频与其他类似的房间游览视频拼接，以创建一个任意长且连续的视觉流。该任务与语言领域常用于压力测试 LLM 长上下文能力的“大海捞针”（NIAH）测试相似~[liu2023lost]。类似的 NIAH 设置也已被提议用于长视频评估~[zhao2024needle,wei2025videorope,hu2025nemo]。然而，与插入无关文本段落或帧的基准测试不同，\vso 通过帧内编辑保持了“针”的真实性。它通过要求顺序回忆进一步增加了挑战，这实际上是一个多跳推理任务，并且在视频长度上可以任意扩展。为了全面评估模型在不同时间尺度上的表现，该基准测试提供了五种时长：10、30、60、120 和 240 分钟。有关 \vso 基准测试构建的更多细节见 \cref{appx:vsisuper}。

\paragraph{\vsisuper Count：视角和场景变换下的持续计数。}
这里我们测试 MLLM 在长程空间视频中持续积累信息的能力。
为了构建 \vsc，我们拼接了来自 VSI-Bench~[yang2024think] 的多个房间游览视频剪辑，并要求模型计算所有房间中目标物体的*总数*（见 \cref{fig:mem-count-illustration}）。
这种设置具有挑战性，因为模型必须处理视角切换、重复观察和场景转换，同时保持一致的累积计数。对于人类来说，计数是一个直觉且可泛化的过程。一旦理解了“一”的概念，将其扩展到更大的数量是自然的。相比之下，正如我们稍后证明的，当前的 MLLM 缺乏真正的空间认知，过度依赖学习到的统计模式。

除了标准评估（即在视频结束时提问）外，我们还在多个时间戳查询模型，以评估其在流式设置中的表现，在这种设置中 \vsc 的正确答案随时间动态演变。为了检验长期一致性，\vsc 包含四种视频时长：10、30、60 和 120 分钟。对于这项定量任务，我们报告使用平均相对准确率（$\mathcal{MRA}$）指标的结果，这与 VSI-Bench 评估协议一致~[yang2024think]。

\paragraph{前沿模型在 \vsisuper 上面临挑战。}
为了测试 \vsisuper{} 是否对前沿 MLLM 构成真正挑战，我们评估了最新的 Gemini-2.5-Flash~[team2024gemini]。如 \cref{tab:gemini_2.5_flash_results} 所示，该模型在处理两小时视频时达到了其上下文限制，尽管其上下文长度为 1,048,576 个 token。这突显了视频理解的*开放性*本质，其中连续流实际上需要“无限输入，无限输出”的上下文，并且可以任意增长，这表明仅靠扩展 token、上下文长度或模型规模可能并不足够。尽管我们的基准测试是合成的，但它反映了空间超感知中的一个真实挑战：人类能够毫不费力地整合并保留长达数小时或数年的持续感官体验中的信息，而目前的模型缺乏类似的持续感知和记忆机制。 
Gemini-2.5-Flash 在关注语义感知和语言理解的视频基准测试（如 VideoMME~[fu2025video] 和 VideoMMMU~[hu2025video]）上表现强劲，达到了约 80\% 的准确率。然而，即使对于 \vsisuper 中完全处于其上下文窗口内的 60 分钟视频，其在 \vso 和 \vsc 上的表现仍然有限——分别仅为 41.5 和 10.9。如 \cref{fig:gemini_vsc_prediction_distribution} 所示，模型预测的物体数量未能随视频长度或真实的物体数量而增长，而是饱和在一个较小的常数值上，这表明其计数能力缺乏泛化性，且依赖于训练分布的先验。

\paragraph{\vsisuper 如何挑战当前范式。}
虽然任务设置很简单，但 \vsisuper 带来的挑战超出了单纯的空间推理，揭示了当前 MLLM 范式的根本局限性。

\finding{}{\faBookmark~~\vsisuper 任务挑战了“仅靠扩展规模就能保证进步”的信念。}

%First, these tasks question the belief that scaling alone guarantees progress. 
通过允许模拟流式认知动态的任意长视频输入，\vsisuper 被有意设计为超过任何固定的上下文窗口。这种设计表明，逐帧标记化（Tokenization）和处理在计算上不太可能作为一种长期解决方案。
% In contrast, 
相比之下，人类通过有选择地关注和保留仅一小部分的感官输入，能够高效且自适应地解决此类问题 (每只眼睛的 600 万个视锥细胞可以发送约 1.6 Gbits/s 的数据，但大脑仅使用 10 bits/s 来引导行为~[koch2006much,zheng2025unbearable])，且通常是无意识的~[fei2007we,von1867handbuch]。这种预测性和选择性机制是人类认知的核心，在当前的 MLLM 中仍然缺失，但它是预测性世界模型的基础。

\finding{}{\faBookmark~~\vsisuper 任务要求在测试时泛化到新的时间和空间尺度。}
例如，\vsc 要求在任意长的视频中进行计数，类似于理解计数概念的人类可以将其扩展到任何数字。关键不在于维持一个极长的上下文窗口——人类不会保留长期视觉体验中的每一个视觉细节——而是在于学习计数过程本身。预测性感知通过将连续的视觉流*分割*成连贯的事件，利用“惊讶”时刻来施加时间结构，从而促进了这一点。这种分割作为一种分而治之的机制，允许模型在动态变化的场景中决定何时开始、持续或重置行为。

总之，这些挑战跨越了计算效率、泛化能力以及诸如无意识推理和预测性感知等认知机制，呼吁一种范式转变。未来的模型不应仅仅依赖于扩展数据、参数或上下文长度，而应该学习内部世界模型，使其能够在随空间和时间无限展开的视觉世界中进行感知和预测。为了进一步论证这一范式转变的必要性，下一节将探讨在当前范式下，通过改进工程和有针对性的数据策展，仍能在多大程度上取得进展。我们评估了现有的 MLLM 框架是否可以进行调整，以应对 \vsisuper 带来的挑战。这些努力虽然受限于当前框架的约束，但对于构建下一代空间超感知模型的数据和实证基础是不可或缺的。

## 当前范式下的空间感知
\label{sec:limits}
如前一节所示，Gemini-2.5-Flash 在空间感知任务上表现不佳（见 \cref{tab:gemini_2.5_flash_results}）。这一观察结果提出了一个关键问题：*受限的空间感知是否仅仅是一个数据问题？* 这是一个值得探讨的问题，因为当前的视频 MLLM 在训练期间并没有明确优先考虑以空间为中心的视频，而且现有的预训练和后训练设计是否适合我们的目标任务仍不确定。我们首先通过一系列架构和训练改进来增强 Cambrian-1~[tong2024cambrian]，从而建立一个更强大的图像 MLLM 作为基座模型（\cref{sec:limits:upgrade-cambrian-1}）。接着，我们构建了一个大规模、以空间为中心的指令微调数据集 \vsidata（\cref{sec:limits:data}）。该数据集由多样化的来源策划而成，并经过精心标注。由于此类数据目前在公开领域并不存在，\vsidata 旨在为空间感知提供坚实的数据基础。最后，通过改进的训练方案（\cref{sec:limits:training_recipe}），我们推出了具有空间对齐能力的 \cambrianS 模型系列（\cref{sec:limits:cambrian-s}）。

\cambrianS{} 模型系列在现有的空间推理基准测试（如 VSI-Bench~[yang2024think]）上展示了强大的性能，并为基座模型设计、数据策划以及空间超级感知的训练策略提供了宝贵的见解。然而，尽管取得了这些进展，这种方法并不能直接解决 \vsisuper 的持续感知挑战（\cref{sec:limits:resuls}）；相反，它提供了一个重要的基础，激发了（\cref{sec:predictive-sensing}）中引入的新范式。

### 基座模型训练：升级版 Cambrian-1
\label{sec:limits:upgrade-cambrian-1}
我们首先开发一个基于图像的 MLLM 基座模型，因为鲁棒的语义感知构成了高层空间认知的基础。我们遵循 Cambrian-1~[tong2024cambrian] 的两阶段训练流水线。我们将视觉编码器升级为 SigLIP2-SO400m~[tschannen2025siglip]，并将语言模型升级为指令微调后的 Qwen2.5~[yang2024qwen2.5]。
对于视觉-语言连接器，我们采用了简单的两层 MLP，这主要是出于计算效率的考虑。Cambrian-1 的其他训练组件，包括超参数和数据方案，保持不变。详细的实现细节见 \cref{appx:impl_details}。

众所周知，数据质量和多样性在 MLLM 的训练中起着关键作用~[tong2024cambrian,mckinzie2024mm1]。我们假设 VSI-Bench~[yang2024think] 上的性能差距主要源于当前指令微调数据集~[zhang2024video,cui2025comprehensive]中缺乏高质量的空间对齐数据。为了填补这一空白，我们构建了 **\vsidata**，这是一个旨在提高视觉-空间理解能力的大规模指令微调数据集。

\paragraph{数据策划与处理。}
我们从多样化的数据来源和类型（即仿真和真实）构建 \vsidata。有关每个数据集的数据来源以及视频、图像和问答对数量的统计信息，请参见 \cref{tab:data_statistics}。我们发现，与从单一来源获得的同等规模数据集相比，这种方法产生的数据集要鲁棒得多。下面，我们详细介绍数据处理过程。

[leftmargin=16pt,topsep=0pt,itemsep=0.1em]

    - *有标注的真实视频。*
        多模态视觉-空间推理依赖于对 3D 几何和空间关系的深刻理解。遵循 VSI-Bench，我们重新利用了现有室内扫描和第一人称视频数据集的训练划分，这些数据集提供了 3D 实例级标注，包括 S3DIS~[armeni20163d]、ScanNet~[dai2017scannet]、ScanNet++~V2~[yeshwanth2023scannet++]、ARKitScenes~[dehghan2021arkitscenes] 和 ADT~[pan2023aria]。对于每个数据集，标注被整合到一个元信息文件中，该文件记录了场景级属性，如按类别分类的对象数量、对象边界框、房间尺寸及相关元数据。然后，自动实例化问题模板以生成相应的问题。
    - *仿真数据。*
        由于 3D 标注数据的可用性有限，仅从真实的标注视频构建大规模且多样化的 3D 标注 SFT 数据集具有挑战性。遵循 SIMS-V~[brown2025simsv]，我们利用具身仿真器程序化地生成具有空间对齐的视频轨迹和问答对，在具有多样化布局、对象配置和视觉外观的 ProcTHOR~[deitke2022ProcTHOR] 场景中渲染了 625 个视频遍历。
        我们将同样的方法应用于 Hypersim~[roberts2021hypersim]，从 461 个室内场景中采样了 5,113 张图像。利用实例级边界框，我们生成了与真实标注视频设置一致的问答对。
    
    - *无标注的真实视频。*
        虽然网络来源的视频缺乏明确的标注，但它们在室内环境类型、地理区域和空间布局方面提供了丰富的多样性。我们从 YouTube 收集了大约 1.9 万个房间游览视频，并额外整合了来自机器人学习数据集（包括 Open-X-Embodiment~[o2024open] 和 AgiBot-World~[bu2025agibot]）的视频。由于这些视频不包含构建空间指令微调数据所需的 3D 标注，我们开发了一个伪标注流水线。如 \cref{fig:data_collect_pipeline} 所示，我们对视频帧进行下采样和过滤，应用目标检测~[liu2024grounding]、分割模型~[ravi2024sam] 和 3D 重建模型~[wang2025vggt]，按照 SpatialVLM~[chen2024spatialvlm] 的方法生成伪标注图像。我们选择在图像层面而非整个视频层面生成标注，因为从识别和重建模型获得的完整视频伪标注往往噪声过大，不适合训练。

\paragraph{问题类型定义与模板增强。}
我们在时空分类体系中定义了 12 种问题类型，以构建一套全面且多样化的指令微调问题。我们定义了五种主要问题类型——\texttt{size}（尺寸）、\texttt{direction}（方向）、\texttt{count}（计数）、\texttt{distance}（距离）和 \texttt{appearance order}（出现顺序）——根据~[yang2024think]，这些类型被广泛归类为测量 \texttt{configuration}（配置）、\texttt{measurement}（测量）或 \texttt{spatiotemporal}（时空）能力。
除了 \texttt{appearance order} 类型外，每个问题类别都包含相对和绝对变体，反映了这些互补推理形式在视觉-空间理解中的重要性~[yang2024think]。例如，对于 \texttt{size}，我们会询问两个物体之间的尺寸比较（*相对*）以及物体的度量尺寸（*绝对*）。

\paragraph{\vsidata 数据源消融实验。}
为了评估我们提出的 \vsidata 数据集的有效性，我们进行了一项消融研究。我们使用 LLaVA-Video-178K~[zhang2024video] 中的部分视频指令微调样本，对 ~\cref{sec:limits:upgrade-cambrian-1} 中描述的改进版 Cambrian-1 MLLM 进行微调。该模型在 \cref{tab:vsidata_source_ablation} 中作为*基准*。
通过在单个数据集以及它们的组合上微调模型，来评估每个数据源的贡献。\vsidata Full Mix 在视频空间推理任务上实现了最高的整体性能，优于基准模型和所有单一数据源模型。**所有数据源在微调后都做出了积极贡献**，尽管它们的有效性各不相同。
\finding{}{\faBookmark~~数据有效性排序为：标注的真实视频 $>$ 模拟数据 $>$ 伪标注图像。}
这表明，对于空间推理而言，视频本质上比静态图像包含更多信息，因为仅在视频数据上进行训练就能在基于视频和图像的空间推理基准测试上均取得优异性能。这些发现支持了这样一种直觉：视频的时间连续性和多视图多样性是开发鲁棒空间表示的关键。

### 空间感知的后训练方案
\label{sec:limits:training_recipe}

我们进一步分析并消融了我们的视频指令微调流水线，重点关注预训练基础视频模型和指令微调数据集混合策略的作用。如 \cref{tab:training_pipeline} 所示，我们从四个代表视频理解能力逐步增强的*基础模型*开始：

[nosep, leftmargin=*]
    - **A1** 仅在 Cambrian-1 对齐数据上进行图像-文本对齐训练。由于语言模型在训练期间被冻结，因此它与基础 QwenLM 相同。
    - **A2** 在 A1 的基础上进行了图像指令微调，本质上就是我们改进的 Cambrian-1。
    - **A3** 从 A2 初始化，并在 429K 视频指令微调数据上进行了微调。
    - **A4** 从 A2 初始化，并在 3M 视频指令微调数据上进行了微调。

然后，我们使用两种不同的数据方案微调这些模型：
(1) 仅使用 \vsidata，以及 (2) 将 \vsidata 与类似数量的通用视频指令微调数据混合。

\finding{}{\faBookmark~~具有更多通用视频数据暴露的更强基础模型在 SFT 后会产生更好的空间感知能力。}
如 \cref{tab:training_pipeline} 所示，使用更强大的基础模型进行有监督微调（SFT）可以增强空间理解能力，这些模型在 VideoMME~[fu2025video] 和 EgoSchema~[mangalam2023egoschema] 等通用视频基准测试中表现良好。这凸显了在基础模型训练期间广泛接触通用视频数据的重要性。

\finding{}{\faBookmark~~混合通用视频数据可以防止域内 SFT 导致的泛化能力损失。}
此外，虽然仅在 \vsidata 上进行域内 SFT 在 VSI-Bench 上实现了最高性能，但它会导致通用视频基准测试性能的明显下降。然而，通过在包含通用视频的混合数据上进行训练，可以有效缓解这种性能下降。

### \cambrianS：具备空间定位能力的 MLLM
\label{sec:limits:cambrian-s}

基于之前所有的见解，我们开发了 **\cambrianS**，这是一个具有不同 LLM 规模（0.5B、1.5B、3B 和 7B 参数）的空间定位模型系列。
这些模型是通过一个专门设计的四阶段训练流水线构建的，该流水线旨在先建立通用的语义感知，然后培养专门的空间感知技能，如图 \cref{fig:training_stage_simple_illustration} 所示。

前两个阶段遵循 Cambrian-1 框架以培养强大的图像理解能力。在第 3 阶段，我们通过在 \osmix 上进行通用视频指令微调将模型扩展到视频领域，\osmix 是一个由 300 万个样本组成的精选数据集（详细组成见 \cref{fig:osmix_sources}）。这一阶段在引入专门技能之前，为通用视频理解奠定了坚实的基础。在最终且关键的第 4 阶段，模型进行空间感知训练。在这里，我们按照 \cref{tab:training_pipeline} 中描述的设置，在结合了我们专门的 \vsidata 与第 3 阶段所使用的通用视频数据比例子集的混合语料库上对模型进行微调。完整的训练细节在 \cref{appx:cambrians_training_recipe} 中提供。

\paragraph{提升的空间认知。}
如 \cref{tab:final_results} 所示，我们的模型在视频视觉-空间理解方面达到了最先进的性能。\cambrianS-7B 在 VSI-Bench 上达到了 67.5\%，显著优于所有开源模型，并以超过 16 个百分点的绝对优势超过了闭源模型 Gemini-2.5-Pro。
由于我们在本节中的工作可以看作是一次数据缩放尝试，一个自然的问题是：*性能的提升仅仅是因为更广泛的数据覆盖（包括更多样化的视觉配置和问答对），还是模型确实发展出了更强的空间认知能力？* 首先，我们要强调，\vsidata 与基准数据集之间不存在数据重叠。尽管一些数据集来源于相同的源（例如来自 ScanNet），但我们仅使用了训练集，而基准测试使用的是验证集和测试集。此外，我们观察到空间推理泛化能力的明显迹象。例如，在极具挑战性的“路径规划”子任务中，由于标注成本高，其问题类型在 \vsidata{} 中并不存在，但 \cambrianS-7B 仍表现强劲，并且随着模型规模的增加展现出明显的缩放行为（见 \cref{tab:vsibench_subtask_results}）。

此外，即使在较小的模型规模下，我们的训练方法也证明是高效的：我们最小的 *0.5B 模型* 在 VSI-Bench 上的表现与 Gemini-1.5 Pro 相当。重要的是，这种对空间推理的强调并没有以牺牲通用能力为代价：\cambrianS{} 在 Perception Test~[patraucean2023perception] 和 EgoSchema~[mangalam2023egoschema] 等标准视频基准测试上继续提供具有竞争力的结果（完整结果见 \cref{tab:video_benchmarks_results}）。

\finding{}{\faBookmark~~\cambrianS{} 实现了最先进的空间感知性能，对未见过的空间问题类型具有鲁棒的泛化能力，同时在通用视频理解方面保持了竞争力。}

\paragraph{在 VSI-Bench-Debiased 上的稳健空间推理。}
最近的一项研究~[brown2025shortcuts] 表明，模型在进行空间推理任务时可能会依赖强大的语言先验。例如，当被要求估算桌子的长度时，模型可能会利用关于典型桌子尺寸（例如 120–180 厘米）的自然世界知识，而不是分析视觉证据。
为了调查 \cambrianS{} 是否学会了通过视觉进行推理，我们在 VSI-Bench-Debiased~[brown2025shortcuts] 上对其进行了评估，该基准测试专门设计用于通过去偏来消除语言捷径。如 \cref{tab:final_results} 所示，尽管与标准 VSI-Bench 相比，性能下降了约 8\%，但我们的模型仍然优于闭源对应模型，证明了稳健的视觉-空间推理能力，并确认了我们的训练超出了基于语言的学习范畴。

\paragraph{VSI-Super 上的结果：连续空间感知的局限性。}

尽管 \cambrianS 在 VSI-Bench 的简短、预分段视频的空间推理任务上表现强劲，但它并不具备良好的连续空间感知能力。这一局限性体现在两个方面。首先，其性能在长视频上显著下降。
如 \cref{tab:cams7b_on_vsi_super} 所示，在流式设定下以 1 FPS 采样率在 \vsisuper{} 上进行评估时，随着视频长度从 10 分钟增加到 60 分钟，得分从 38.3\% 稳步下降到 6.0\%，且模型在超过 60 分钟的视频上完全失效。
其次，模型难以泛化到新的测试场景。虽然模型在多房间房屋参观视频上进行了训练，但它无法处理仅多出几个房间的未见过示例。这不仅仅是上下文长度的问题：即使在完全处于模型上下文窗口内的 10 分钟短视频上，性能也会下降。这些结果强调，在当前的 MLLM 框架内，单纯的数据驱动方法，无论投入多少数据或工程努力，都面临着根本性的限制。解决这些局限性需要向 AI 系统进行范式转变，使其能够主动建模和预测世界，同时更有效地组织其经验，我们将在接下来的部分中对此进行探索。

## 预测性感知作为一种新范式
\label{sec:predictive-sensing}

Gemini-2.5-Flash（\cref{tab:gemini_2.5_flash_results}）和 \cambrianS{}（\cref{tab:cams7b_on_vsi_super}）在 \vsisuper{} 上的性能均急剧下降，揭示了一个根本性的范式差距：仅靠扩展数据和上下文长度不足以实现超感知（supersensing）。
我们提出将*预测性感知*（predictive sensing）作为前进的方向，即模型学习预测其感知输入，并构建内部世界模型来处理无界视觉流。
%
这一设计灵感源于人类认知理论。与当前对整个数据流进行标记化（tokenize）和处理的视频多模态模型不同，人类的感知（和记忆）具有高度选择性，仅保留一小部分感知输入~[von1867handbuch,millidge2022predictive,hohwy2013predictive,rao1999predictive]。
大脑持续更新内部模型以预测即将到来的刺激，压缩或丢弃那些不提供新信息的、可预测的输入~[clark2013whatever,friston2010free]。
相反，违反预测的意外感知信息会产生“惊讶感”（surprise），并驱动更多的注意力分配和记忆编码~[schultz2000neuronal,gershman2017computational,kennedy2024prediction]。
%
我们通过一种自监督的下一潜空间帧预测（next-latent-frame prediction）方法初步实现了这一概念（\cref{sec:predictive-sensing:lfp}）。
由此产生的预测误差可作为两种关键能力的控制信号：选择性保留重要信息的存储管理（\cref{sec:case_study_1_vso}），以及将无界流划分为有意义区块的事件分割（\cref{sec:case_study_2_vsc}）。
通过在 \vsisuper{} 上的两个案例研究，我们证明了该方法显著优于强大的长上下文和流式视频模型基线。

### 通过潜空间帧预测实现预测性感知

\label{sec:predictive-sensing:lfp}
我们通过一个轻量级的自监督模块——潜空间帧预测（Latent Frame Prediction, LFP）头来实现预测性感知范式，该模块与主要的指令微调目标共同训练。这是通过按如下方式修改第四阶段训练方案实现的：

[nosep, leftmargin=*]
    - **潜空间帧预测头。**
    我们引入了一个 LFP 头，这是一个与语言头并行运行的两层 MLP，用于预测后续视频帧的潜空间表征。该架构如图 \cref{fig:lfp_framework} 左上角所示。
    - **学习目标。**
    为了优化 LFP 头，我们引入了两个辅助损失：均方误差（MSE）和余弦距离，用于衡量预测的潜空间特征与下一帧的真值特征之间的差异。一个权重系数用于平衡 LFP 损失与主要的指令微调下一标记预测（next token prediction）目标。
    - **LFP 训练数据。**
    我们使用来自 \vsidata 的 29 万条视频子集扩充了第四阶段数据，专门用于 LFP 目标。与指令微调不同，这些视频以 1 FPS 的恒定速率采样，以确保潜空间帧预测的时间间隔均匀。

在此修改后的第四阶段微调期间，我们以端到端的方式联合训练连接器（connector）、语言模型以及语言和 LFP 头，同时保持 SigLIP 视觉编码器冻结。所有其他训练设置与原始第四阶段配置保持一致。
为简便起见，在随后的实验中，我们仍将使用 LFP 目标联合优化的模型称为 \cambrianS{}。

\paragraph{推理：通过预测误差估算惊讶感。}
在推理过程中，我们利用训练好的 LFP 头来评估每个输入视觉感知的“惊讶感”。在心理学中，这一框架通常被描述为预期违背（Violation-of-Expectation, VoE）范式~[burgoon1988nonverbal]。具体而言，在推理期间，视频帧以恒定采样率输入 \cambrianS{}。除非另有说明，以下实验中的视频在输入模型之前均以 1 FPS 采样。
当模型接收到输入的视频帧时，它会持续预测下一帧的潜空间特征。然后，我们测量模型预测与该输入帧的实际真值特征之间的*余弦距离*。该距离作为惊讶感的定量衡量指标：数值越大，表示与模型学到的预期的偏差越大。这种惊讶感得分作为一个强大的、自监督的引导信号，用于随后探索的下游任务。

### 案例研究 I：用于 \vsisuper{
 回答的惊讶感驱动存储管理系统}\label{sec:case_study_1_vso}
目前大多数 MLLM 对所有视频帧一视同仁，存储每一帧而不进行选择性压缩或遗忘，这限制了效率和可扩展性。在本案例研究中，我们探索为 MLLM 增加一个**惊讶感驱动的存储管理**框架，以支持对长视频进行持续的空间感知问答。
我们展示了通过惊讶感引导的压缩，\cambrianS{} 能够保持一致的准确度和稳定的 GPU 显存占用，且与视频长度无关。

\paragraph{惊讶感驱动的存储管理系统。}
我们的存储管理系统根据对“惊讶感”的评估动态地压缩和整合视觉流。
如 \cref{fig:sor_memory_design}-a 所示，我们使用固定窗口大小的滑动窗口注意力对输入帧进行编码。潜空间帧预测模块随后衡量“惊讶程度”并将其分配给每帧的 KV 缓存。惊讶程度低于预定义阈值的帧在存入长期存储之前会经过 $2\times$ 压缩。
为了保持稳定的 GPU 显存占用，该长期存储的大小受整合函数的限制，该函数同样基于*惊讶感*运行：根据惊讶感得分丢弃或合并帧（见 \cref{fig:sor_memory_design}-b）。
% 当达到此限制时，整合模块将较不令人惊讶的帧与相邻帧合并（见 \cref{fig:sor_memory_design} (b)）。
最后，在收到用户查询时，系统通过计算查询与存储帧特征之间的余弦相似度，从长期存储中检索出前 $K$ 个最相关的帧（见 \cref{fig:sor_memory_design}-c）。

\paragraph{结果。}
我们在 \vso{} 基准测试中对比了配备和不配备惊讶感存储系统的 \cambrianS{}，以及两个先进的闭源模型：Gemini-1.5-Flash~[team2024gemini] 和 Gemini-2.5-Flash~[comanici2025gemini]。
如 \cref{fig:accuracy_on_vlo} 所示，\cambrianS{}（含存储）在所有视频长度上的表现均优于 Gemini-1.5-Flash 和 \cambrianS{}（不含存储），证明了其在不同视频时长下一致的空间感知性能。尽管 Gemini-2.5-Flash 在一小时内的视频中表现强劲，但它无法处理更长的输入。
除了保持高准确度外，\cambrianS{}（含存储）在不同视频长度下也保持了稳定的 GPU 显存使用（\cref{fig:gpu_mem_on_vlo}）。
这表明基于惊讶感的存储能有效地压缩冗余数据而不丢失关键信息。我们在 \cref{tab:comparisons_with_existing_long_video_methods} 中加入了两个长视频基线模型 MovieChat~[song2024moviechat] 和 Flash-VStream~[zhang2024flash] 进行对比。

\paragraph{惊讶感度量的消融实验。}
我们基于惊讶感的存储系统的核心是衡量惊讶感的机制，它决定了如何在不假设任何关于未来查询的先验知识的情况下，以被动感知的方式压缩或整合帧。在这里，我们将我们的设计（以预测误差作为惊讶感）与另一个直观的基线进行比较：相邻帧的视觉特征相似度。具体来说，我们使用 SigLIP2 作为视觉编码器，直接比较两个相邻帧之间的帧特征差异（余弦距离）。如果差异超过阈值，我们将后一帧视为惊讶帧。
我们在所有 \vso{} 变体上比较了这两种方法。
对于每个 \vso{} 时长，除了惊讶感阈值外，我们保持实验设置完全相同，并对两种方法的阈值都进行了调整。
如 \cref{fig:ablation_on_vso} 所示，在不同的视频时长下，使用预测误差作为惊讶感度量始终优于相邻帧相似度。\finding{}{\faBookmark~~预测性感知在建模视频数据时空动态方面，比基于逐帧特征的静态相似性度量提供了一种更具原则性的方法。}
虽然我们目前的系统采用了一个简单的预测头作为初步原型，但未来集成功能更强大的世界模型可能会产生更丰富、更可靠的惊喜信号，最终推动空间超感知领域的广泛进步。

 计数的惊喜驱动连续视频分割}\label{sec:case_study_2_vsc}
虽然 \vso{} 侧重于评估 MLLM 的长期观察和召回能力，但超感知更具挑战性的测试将涉及测试模型理解其感官输入、在不同环境中导航以及执行累积、多跳推理的能力。例如，模型可能需要在一个环境中完成一项任务，移动到另一个环境，并最终整合所有经历的信息以做出最终决策。

\end{figure}

\paragraph{惊喜驱动的事件分割。}
“事件”可以被理解为一段时空连贯的经历片段~[kurby2008segmentation]。在空间超感知的语境下，一个事件对应于置身于特定空间并感知其环境的连续经历。这一定义强调，真实的感官体验通常被组织成局部连贯的片段——即感知、空间和时间特征保持相对稳定或一致的情节。因此，事件分割是根据这种连贯性的变化将连续的感官输入流解析为离散、有意义单元的过程。这种分割对于推理和行为至关重要~[dominey2021narrative]：它允许智能体（生物或人工）构建经历的结构化表示，检测发生重大变化的边界，并相应地更新对环境的预测。最近的研究强调，预测误差和工作记忆/上下文的变化是驱动分割的两种可能机制~[nolden2024prediction,shim2024generating]。

在 \vsisuper{} 计数（VSC）基准测试中，我们研究了一个简单的设置，其中惊喜（surprise）被用于分割连续的视觉输入，将场景变化识别为划分视频流为*空间连贯*片段的自然断点。这种方法也与人类解决问题的方式相呼应：在广阔区域内清点物体时，人们通常会一次专注于一个区域，然后再合并结果。这种行为还与“门口效应”~[radvansky2011walking]有关，即穿过门口或进入新房间会在记忆中创建一个自然的边界。
%
如 \cref{fig:vsi_long_count_framework} 所示，模型在事件缓冲区中持续累积帧特征。当检测到高惊喜帧时，缓冲区特征被汇总以产生片段级答案，并清空缓冲区以启动新片段。这一循环重复直到视频结束，之后所有片段答案被聚合以形成最终输出。

\paragraph{结果。}
Gemini-1.5-Flash 在 \vsc{} 上表现出接近零的性能（\cref{fig:count-results}），显示了该任务的难度。虽然 Gemini-2.5-Flash 在 10 分钟视频上取得了更好的结果，但其性能在更长的视频中迅速下降。
相比之下，\cambrianS{} 使用的惊喜驱动事件分割方法（*w/* Surprise Seg.）在所有视频长度上均实现了更高且更稳定的性能。当使用地面真值场景转换对视频进行分割时（即 \cambrianS{} *w/* GT Seg.），性能进一步提升，代表了约略的性能上限。

\cref{fig:count_pred_distribution} 中的深入分析表明，Gemini-2.5-Flash 的预测被限制在有限的范围内，且不会随着视频中出现更多物体而扩展。
相比之下，\cambrianS{}（*w/* Surprise Seg.）产生的计数虽然尚未完全准确，但与真实物体数量表现出更强的相关性，表明其具有更好的泛化能力。

我们将我们的惊喜驱动方法与使用相邻帧特征相似性的基准方法进行了比较（\cref{fig:count-comparison}）。

\paragraph{流式设置下的评估。}
由于 \vsc{} 中的正确答案在整个视频过程中不断演变，我们创建了一个流式问答设置，在 10 个不同的时间戳提出相同的问题。最终性能是对所有查询结果的平均。
%
我们针对定位为实时视觉输入的商用 MLLM 进行了基准测试。如 \cref{fig:vsc_streaming_results} 所示，虽然 Gemini-Live 和 GPT-Realtime 旨在用于流式场景，但它们在 10 分钟视频上的 MRA 低于 15\%，且在 120 分钟流中的性能下降到接近零。然而，\cambrianS{} 表现出更强的性能，在 10 分钟流中达到 38\% 的 MRA，并在 120 分钟时保持在 28\% 左右。

\paragraph{总结。} 在 VSR 召回和 VSC 计数任务中，通过惊喜驱动的记忆和事件分割进行的预测性感知，使 \cambrianS{} 能够克服 \cref{sec:limits} 中描述的固定上下文限制。虽然这仍是一个初步原型，但它突显了构建不仅能“看”而且能预测、筛选和组织经历的 AI 系统的潜力。这类系统超越了帧级问答，转而构建隐式世界模型，以支持更深层的空间推理，扩展到无限的时间跨度，并实现能够媲美并最终超越人类视觉智能的超感知。

## 相关工作
\label{sec:related_work}

\paragraph{视频多模态大语言模型} 预训练大语言模型（LLMs）强大的语言理解能力~[brown2020language,touvron2023llama,bai2023qwen,touvron2023llama2]，结合作为特征提取器的视觉基础模型的表示能力~[radford2021learning,zhai2023sigmoid,tschannen2025siglip,he2022masked,fan2025scaling]，推动了将这些模型扩展到文本之外以实现对视觉内容语义感知的重大进展，且最初主要集中在图像领域~[hurst2024gpto,liu2023visual,li2024llava,bai2023qwenvl,tong2024cambrian,team2023gemini,chen2024internvl,wang2024qwen2vl,li2023blip2]。这一势头激发了对基于视频的多模态大语言模型（MLLMs）日益增长的研究~[li2024llama,li2024llava,zhang2024video,song2024moviechat,bai2025qwen2,zhu2025internvl3,zhang2023video,li2023videochat,zohar2024apollo,marafioti2025smolvlm]，这被视为将多模态智能与具身智能体等现实世界应用相结合的关键一步~[kim2024openvla,yang2024virl]。正如本文始终强调的，开发一个真正强大的超感系统需要重新思考几个核心方面，包括如何进行基准测试、什么是合适的数据、哪些架构设计最有效，以及什么样的建模目标最能与系统目标保持一致。

\paragraph{流式视频理解}
视频是连续且可能无限的视觉信号流。虽然人类可以毫不费力地处理它，但其无界性对视频 MLLM 构成了挑战，因为 token 长度会随视频时长而增加，导致计算和存储成本不断上升。最近的研究探索了几种方法来解决这个问题：*高效架构设计*。自注意力机制的二次方成本使得处理长视频变得困难。最近的方法~[li2024videomamba,ren2025vamba]使用了更简单、更快速的架构~[wang2020linformer,gu2023mamba,katharopoulos2020transformers]，这些架构能够减少计算量并更好地处理长输入。*上下文窗口扩展*。预训练 LLM 中固定的上下文长度限制了它们对长期内容的理解。最近的研究~[chen2024longvila,zhang2024long,chen2025scaling]通过精心的系统设计扩展了这一窗口，使模型能够处理并推理更长的视频序列。*检索增强的视频理解*。为了处理长视频，一些方法仅从较大的集合中检索最相关的片段~[korbar2024text,pan2025timesearch,wang2024videoagent]，并将其作为后续分析的上下文。*视觉 token 减少或压缩*。其他方法通过减少帧间或帧内的视觉 token 来缩短输入~[shen2024longvu,li2024videochat,jiang2025token,li2025lion, chai2024auroracap]，从而更容易处理长视频序列。虽然这些方法提高了性能，但它们在很大程度上将连续视频视为类似于文本的标准序列建模问题。我们认为，未来的 MLLM 应该像人类一样，通过构建内部预测模型来高效处理连续视觉流。

\paragraph{视觉空间智能}
从视觉输入中理解空间关系对于感知物理世界并与之交互至关重要。随着多模态模型变得更具物理落地性，对空间智能的关注度激增，催生了新的基准测试~[yang2024think,ramakrishnan2024does,yin2025spatial,majumdar2024openeqa,yeh2025seeing,li2025sti,xu2025multi,team2025gemini]以及专注于增强模型空间推理能力的研究~[yang2025mindjourney,ma2025spatialreasoner,ouyang2025spacer,du2024embspatial,chen2024spatialvlm,cheng2024spatialrgpt,cai2024spatialbot,liu2024coarsecorrespondenceelicit3d,li2024topviewrs,zhu2024llava3d,ray2025sat]。在本文中，我们通过视频中空间超感的概念来研究视觉空间智能，并探索通过改进数据整理、优化训练策略和引入新范式来增强 MLLM 空间推理的方法。

\paragraph{预测建模} 
学习到的内部预测模型~[craik1967nature,ha2018world]允许智能体表示并模拟其环境的各个方面，从而实现更有效的规划和决策。模型预测控制（MPC）~[garcia1989model]在控制理论中应用了类似的原理，利用内部前向模型来预测未来轨迹并实时选择最优动作。这一概念灵感来源于人类如何形成世界的心理模型~[rao1999predictive,hohwy2013predictive,friston2010free]，以及这些内部表示如何影响行为（例如，*无意识推理*~[von1867handbuch]），它们作为现实的简化抽象，能够实现预测和高效行动。越来越多的研究通过自监督表示学习~[assran2023self,assran2025v]以及文本或动作条件的视频生成~[zhou2024dino,yang2023learning,bar2025navigation,chen2024simple,bai2025whole,garrido2025intuitive]探索了预测建模的思想。在本文中，受人类如何利用内部世界模型高效且有效地处理无界感官输入的启发，我们研究了如何为 MLLM 配备类似的预测感知能力。

## 结论

我们强调了视频中空间*超感*（supersensing）能力的重要性并提出了一个层次结构，认为实现超智能需要 AI 系统超越目前大多数 MLLM 所关注的基于文本的知识和语义感知，进而发展空间认知和预测性世界模型。为了衡量进展，我们引入了 $\vsisuper$，并发现目前的 MLLM 在处理该任务时表现欠佳。为了测试当前的进展是否受限于数据，我们构建了 \vsidata，并在其上训练了我们的空间对齐 MLLM——\cambrianS。尽管 \cambrianS 在标准基准测试中表现良好，但其在 \vsisuper 上的结果揭示了当前 MLLM 范式的局限性。我们设计了预测性感知的原型，利用潜变量帧预测和惊喜度估计来处理无边界视觉流。它提升了 \cambrianS 在 \vsisuper 上的性能，并标志着向空间超感迈出的初步探索。

**局限性** 
我们的目标是提出一个概念框架，鼓励社区重新审视开发空间超感能力的重要性。作为一个长期的研究方向，我们目前的基准测试、数据集和模型设计在质量、规模和泛化能力方面仍存在局限性，且该原型仅作为概念验证。未来的工作应当探索更多样化且具身化的场景，并与视觉、语言和世界模型领域的最新进展建立更紧密的联系。

## \vsisuper{
 基准测试}\label{appx:vsisuper}

### \vsisuper{
 召回}

\paragraph{实现细节。}
为了构建这一基准测试，我们首先采用了来自 VSI-Bench 集合~[yang2024think] 的视频。标注人员挑选视频，并使用 Gemini-2.0-Flash 将精选池中的异常物体手动插入到四个不同的帧中，重点是将物体放置在合理的位置。对于每次插入，标注人员都会记录物体的位置及其出现顺序。然后，我们将这些编辑过的片段与随机采样的未编辑视频结合，生成时长分别为 10、30、60、120 和 240 分钟的最终视频。对于每个时长，我们创建了 60 个视频，每个视频对应一个问题。我们将视频下采样至每秒 1 帧，以确保模型在推理过程中始终能看到编辑后的帧。

\paragraph{可视化。}
我们在 \cref{fig:sor_examples} 中展示了 \vso 视频数据集编辑帧的定性示例。插入的物体在其位置上看起来在视觉上是合理的，这直接归功于我们的高质量标注。

### \vsisuper{
 计数}

\paragraph{实现细节。}
为了构建 \vsisuper{} 计数，我们拼接了来自 VSI-Bench~[yang2024think] 的视频，并累加它们的物体数量以创建新的真值（ground truth）。这一过程需要两个额外的标准化步骤。首先，我们统一了来自不同源数据集（即 ScanNet~[dai2017scannet]、ScanNet++~[yeshwanth2023scannet++] 和 ARKitScenes~[dehghan2021arkitscenes]）的物体类别标签。其次，我们通过重新平衡问答对来解决数据对小物体数量的偏见，从而创建更均匀的计数分布。最终的基准测试包括时长为 10、30、60 和 120 分钟的视频，每个视频配有 50 个相应的问题。与 \vso 不同的是，\vsc{} 中的所有视频都下采样至 24 FPS。

\paragraph{流式设置。} 
对于流式设置，我们在 10 个不同的时间戳重复查询视频中的物体总数。为了在这些查询时间戳构建真值，我们需要确定视频中每个唯一物体首次出现的时间。为了找到这些出现时间，我们使用了 VSI-Bench~[yang2024think] 提出的方法。这允许直接计算任何给定时间戳下的真值物体计数。

### 问题类型定义的细节
\label{appx:vsidata:question_type_definition}

\paragraph{分类法。}
在策划视觉空间智能有监督微调数据集时，一个重要的视角是如何定义问题类型。受 VSI-Bench~[yang2024think] 的启发，我们以更系统的方式扩展了其任务定义。如 \cref{tab:vsidata:question_type_definition} 所示，我们从四个视角区分这些问题类型：

    - \texttt{时空属性：} 我们将问题分为五个不同的时空属性类型：尺寸（比较或测量物体/空间维度）、方向（空间中的方位）、计数（物体的枚举）、距离（物体间的接近程度）以及出现顺序（视频中物体出现的时间序列）。
    - \texttt{相对与绝对：} 当问题涉及多个物体之间的比较时（例如，“哪一个更大？”），被分类为相对问题；当需要特定的测量值或数量时（例如，“以米为单位的高度是多少？”），被分类为绝对问题。这种区别适用于大多数属性类型。
    - \texttt{视角选择：} 该维度捕捉评估空间关系时所采用的视角。问题可能从摄像机视角提出（例如，“从摄像机的视角看，物体是在左边还是右边？”），或者从场景中特定物体的视角提出（例如，“从物体$_2$面对物体$_1$...”）。
    - \texttt{模态：} 问题根据是仅使用静态图像即可回答，还是需要动态视频信息进行分类。某些属性类型（如出现顺序）仅适用于视频，而其他属性（如尺寸）可以在任一模态中提问。

此外，继 VSI-Bench 之后，我们还根据不同的时空特征将我们的问题类型分为三个不同的组（即 *布局 (Configuration)*、*测量 (Measurement)* 或 *时空 (Spatiotemporal)*）。

### 详细的问答对构建流水线

我们在此介绍用于策划 \vsidata{} 的具体流水线。

\paragraph{3D 标注的真实视频。}
对于 3D 标注的真实视频，我们遵循 Thinking in Space~[yang2024think] 建立的惯例。我们首先调研所有包含 3D 实例级标注和视频或全景图像的公开数据集。从这些数据集中，我们提取关键信息，包括*物体计数*、*物体边界框*和*房间尺寸*测量值，然后将其标准化为统一格式。随后，这些结构化信息被整合到增强的问题模板中，以创建配对的问答集。

\paragraph{3D 标注的仿真视频和图像。}
对于本质上包含丰富标注的仿真数据，我们遵循与 3D 标注真实视频类似的程序。对于 ProcTHOR~[deitke2022ProcTHOR]，我们的主要工作是生成具有随机放置智能体的 3D 场景以渲染遍历视频。对于提供图像级而非场景级 3D 标注的 Hypersim~[roberts2021hypersim]，我们利用单张图像及其相应的 3D 标注。在这两种情况下，我们都提取必要的信息，将其转换为我们设计的统一格式，并按照用于 3D 标注真实视频的相同方法，将其整合到增强的问题模板中。

\paragraph{未标注的网页抓取真实视频。} 对于未标注的网页抓取真实视频，如算法~\ref{algo:vsidata:qa_gen_web_video} 所示，我们实现了一个多阶段处理流水线。我们首先以定期间隔采样帧并过滤掉模糊图像。对于每个有效帧，我们采用开放词汇目标检测器 Grounding-DINO~[liu2024grounding]，并配合预定义的感兴趣类别。当一帧包含足够的有效物体时，我们使用 SAM2~[ravi2024sam] 提取实例级语义掩码。
此外，为了将 2D 图像内容转换为 3D 表示，我们采用 VGGT~[wang2025vggt] 为每张图像提取 3D 点集，并将其与先前生成的实例掩码集成。值得注意的是，我们应用腐蚀算法来细化实例掩码，这减轻了物体边界处不准确的点云估计。该流水线使我们能够从 YouTube 上的约 19,000 个房间游览视频和机器人学习数据集中创建伪标注，在各种房间类型和布局中生成多样化的空间问答对，而无需手动 3D 标注。通过处理单个帧而不是完整视频，我们的流水线确保了更高质量的语义提取和更可靠的重建结果，避免了将重建和语义提取技术应用于整个视频序列时通常遇到的噪声和不一致问题。

### \vsidata 示例

为了更好地展示 \vsidata，我们在 \cref{fig:annotated_real_video_1,fig:annotated_real_video_2,fig:annotated_real_video_3,fig:annotated_simulated_video,fig:annotate_simulated_video_frame,fig:unannotated_image_1,fig:unannotated_image_2} 中提供了定性可视化结果。这些可视化表明，\vsidata 为空间问答有监督微调提供了极佳的多样性和质量。

\begin{algorithm}[!h]
    \small
    \caption{
        **未标注网页抓取视频的问答生成流水线**
    }\label{algo:vsidata:qa_gen_web_video}
    \KwIn{视频序列 $V$, 有效类别列表 $\mathcal{C}_{\text{valid}}$, 无效类别列表 $\mathcal{C}_{\text{invalid}}$, 采样间隔 $\Delta t$, 模糊阈值 $\tau_{\text{blur}}$, 最小物体计数 $\theta_{\text{min}}$, 最小 3D 点数 $\theta_{\text{3D}}$, 腐蚀核 $K_{\text{erosion}}$}
    \KwOut{选定的帧集合 $\mathcal{F}$, 问答对 $\mathcal{Q}$}初始化 $\mathcal{F} \leftarrow \emptyset$, $\mathcal{Q} \leftarrow \emptyset$\;
    $\mathcal{S} \leftarrow \text{SampleFrames}(V, \Delta t)$ \tcp*{以时间间隔 $\Delta t$ 采样帧}
    
    \ForEach{帧 $f \in \mathcal{S}$}{
        \If{$\text{BlurDetection}(f) > \tau_{\text{blur}}$}{
            **continue**\;
        }
        $\mathcal{O} \leftarrow \text{GroundingDINO}(f, \mathcal{C}_{\text{valid}} \cup \mathcal{C}_{\text{invalid}})$ \tcp*{从两个类别列表中检测物体}
        
        \If{$\exists o \in \mathcal{O} : \text{category}(o) \in \mathcal{C}_{\text{invalid}}$}{
            **continue**\;
        }
        
        $\mathcal{O}_{\text{valid}} \leftarrow \{o \in \mathcal{O} : \text{category}(o) \in \mathcal{C}_{\text{valid}}\}$\;
        \If{$|\mathcal{O}_{\text{valid}}| < \theta_{\text{min}}$}{
            **continue**\;
        }
        $\mathcal{M} \leftarrow \emptyset$ \tcp*{初始化掩码集合}
        
        \ForEach{物体 $o \in \mathcal{O}_{\text{valid}}$}{
            $b \leftarrow \text{GetBoundingBox}(o)$\;
            $m \leftarrow \text{SAM2}(f, b)$ \tcp*{使用 SAM2 生成掩码}
            $m' \leftarrow \text{Erode}(m, $K_{\text{erosion}})$ \tcp*{对掩码应用腐蚀操作}
            $\mathcal{M} \leftarrow \mathcal{M} \cup \{m'\}$\;
        }
        
        $\mathcal{P}_{\text{map}} \leftarrow \text{VGGT}(f)$ \tcp*{使用 VGGT 生成 3D 点映射}
        $\mathcal{P} \leftarrow \emptyset$ \tcp*{初始化 3D 点集}
        
        \ForEach{掩码 $m \in \mathcal{M}$}{
            $P \leftarrow \text{ExtractMaskedPoints}(m, \mathcal{P}_{\text{map}})$ \tcp*{提取掩码覆盖的 3D 点}
            \If{$|P_{\text{valid}}| \geq \theta_{\text{3D}}$}{
                $\mathcal{P} \leftarrow \mathcal{P} \cup \{P\}$\;
            }
        }
                
        \If{$|\mathcal{P}| > 0$}{
            $q \leftarrow \text{QAGenerator}(\mathcal{P})$ \tcp*{根据 3D 几何生成问答对}
            $\mathcal{Q} \leftarrow \mathcal{Q} \cup \{q\}$\;
            $\mathcal{F} \leftarrow \mathcal{F} \cup \{f\}$\;
        }
    }
    **返回** $\mathcal{F}$, $\mathcal{Q}$\;
\end{algorithm}
## \cambrianS{
 实现细节}\label{appx:impl_details}

在本节中，我们提供 \cambrianS{} 模型的全面训练细节。

### 模型架构

遵循原始的 Cambrian-1~[tong2024cambrian] 以及大多数 MLLM~[liu2023visual,li2024llava] 的通用做法，我们的模型（包括升级后的 Cambrian-1 和 \cambrianS）集成了一个预训练的视觉编码器、一个作为解码器的预训练语言模型，以及一个连接这两个模态的视觉-语言连接器。具体而言，我们采用 SigLIP2-So400M~[tschannen2025siglip] 作为视觉编码器。该编码器使用多种损失函数的组合进行训练：文本下一标记预测（LocCa~[wan2024locca]）、图像-文本对比（或 sigmoid~[radford2021learning, zhai2023sigmoid]）以及掩码自我预测（SILC~[naeem2024silc]/TIPS~[maninis2024tips]）。对于语言模型，我们使用指令微调后的 Qwen2.5 LLM~[yang2024qwen2.5]。与使用 SVA 进行更深层次视觉-语言融合的 Cambrian-1 不同，我们采用了更简单的 GELU 激活~[dauphin2017language] 的两层 MLP 作为视觉-语言连接器，以在性能和效率之间保持平衡。

### 训练数据混合

正如 \cref{sec:limits:cambrian-s} 中提到的，我们的 \cambrianS 模型通过四个训练阶段进行训练（参见 \cref{fig:training_stage_simple_illustration}）。对于前两个阶段（即视觉-语言对齐阶段和图像指令微调阶段），读者可以参考 Cambrian-1~[tong2024cambrian] 以了解详细的训练数据混合情况。
在第三阶段，我们对图像指令微调后的模型 \osmix 进行微调，而在最后一个阶段，我们通过在 \vsidata 上微调模型来进行空间视频指令微调。
\osmix 是我们策划的视频指令微调数据集，包含约 300 万个视频问答样本，基于一系列开源视频数据集构建（例如 LLaVA-Video~[zhang2024video]、ShareGPT4o~[cui2025comprehensive]、VideoChat2~[li2024mvbench]、MovieChat~[song2024moviechat]、EgoIT~[yang2025egolife]、Perception Test~[patraucean2023perception]、Vript~[yang2024vript]、VideoChatGPT-Plus~[maaz2024videogpt+]、Ego4D~[grauman2022ego4d]、HowTo100M~[miech2019howto100m]、HD-VILA~[xue2022advancing]、HTStep~[afouras2023ht]、TimeIT~[ren2024timechat]、HowToInterlink7M [wang2024cosmo]、GUI-World~[chen2024gui]、Video-Localized-Narratives [voigtlaender2023connecting] 等）。

\paragraph{阶段 3：通用视频指令微调。}

为了使模型具备通用视频理解能力，我们在策划的 \osmix 视频数据和从 Cambrian-7M 采样的图像指令数据的混合数据集上进行视频指令微调。与之前的阶段一样，视觉编码器保持冻结，其余模块进行微调。
对于图像数据，我们重复使用阶段 2 的采样策略。对于视频数据，我们对每个视频均匀采样 64 帧，将其大小调整为 $384\times384$，并进一步将其特征图下采样为 $8\times8$，即每帧 64 个标记。

\paragraph{阶段 4：空间视频指令微调。}

最后一个阶段侧重于通过在我们提出的 \vsidata 上进行微调来增强模型的空间推理能力。为了保持通用的视频和图像理解能力，我们混合了来自 \osmix 的 59 万个视频样本和来自 Cambrian-7M 的 12 万个图像样本。
训练设置与阶段 3 基本一致，但有两个关键变化：（1）我们将每个视频的帧数增加到 128 帧，（2）我们将序列长度扩展到 16,384，这两者都是为了支持更丰富的时序建模。
阶段 3 和阶段 4 的详细配置列于 \cref{tab:cambrian_s_training_setups} 中。

### 基础设施

本文中的所有模型均使用 TPU v4 Pods 和 TorchXLA 框架进行训练。
为了支持大规模视频指令微调——在这种情况下，长序列长度会带来极高的计算和内存成本——我们利用了由 Pallas 实现的 GSPMD~[xu2021gspmd] 和 FlashAttention~[dao2022flashattention]。

GSPMD 是一个专为灵活且用户友好的大规模分布式训练而设计的自动并行化系统。它允许用户编写训练代码，就像针对单个设备编写一样，然后通过最小的改动毫不费力地在数百个设备上进行扩展。我们的训练框架基于 TorchXLA 和 GSPMD，在多个设备上对数据、模型参数、激活值和优化器状态进行分片。这降低了峰值内存使用量并提高了训练吞吐量。

为了适应长序列，我们集成了由 Pallas 支持的 FlashAttention，这显著降低了长上下文输入下的 TPU HBM（显存）使用量。这使我们能够在 TPU v4-512 Pod 上将 7B 模型的输入序列长度扩展到 16,384 个标记。

为了进一步阐明基于图像和基于视频的指令微调对模型最终视频理解能力的各自贡献，我们进行了一系列实验。这些实验在微调阶段采用了不同比例的图像和视频数据，并观察了由此产生的在各种视频基准上的性能趋势。 更具体地说，对于初始的图像 MLLM 训练，我们从 Cambrian-7M 中随机抽取了 1M、4M 和 7M 个图像问答（QA）对来训练不同的模型。随后，对于视频专用微调，我们从 LLaVA-Video-178K（共计约 $\sim$1.6M 个数据样本）中随机抽取了 25\%、50\%、75\% 和 100\% 的视频 QA 对，对这些预训练的图像 MLLM 分别进行纯视频微调。图像指令微调和视频微调的超参数分别如 Table~\ref{tab:improved_cambrian_training_setups} 和 Table~\ref{tab:cambrian_s_training_setups} 所示。Table~\ref{tab:image_video_scale_results} 展示了实验结果，并得出以下观察结论：

    - *在不进行微调的情况下，在视频基准测试上评估时，使用更多图像数据训练的模型并不一定优于使用较少图像数据训练的模型。* 如表中所示，在视频基准测试上的直接评估显示，最初分别在 1M、4M 和 7M 图像数据集上训练的三个模型表现相当。
    - *在视频数据上进行微调通常对在较大图像数据集上预训练的模型有益，但并非普遍适用。* 当所有模型都在 100\% 的视频数据上进行微调时，最初在 7M 图像上训练的模型在 9 个视频基准测试中的 5 个（具体为 HourVideo、VideoMME、EgoSchema、LongVideoBench 和 Perception Test）表现优于其他两个模型。
    - *在训练过程中引入视频数据可以持续提升所有视频基准测试的性能。* 我们观察到，使用视频数据（即使是 25\% 这样的小部分）对基于图像的 MLLM 进行微调，可以提高其在所有评估的视频基准测试中的性能。
    - *增加用于微调的视频数据量并不能保证在所有基准测试中都能获得一致的性能提升。* 虽然视频微调通常是有利的，但某些基准测试（例如 VideoMME、VSI-Bench、Tomato）并没有随着视频数据的增多而显示出进一步的增益。例如，在 VideoMME 基准测试上，使用 100\% 视频数据微调的模型表现与仅使用 25\% 视频数据微调的模型相当。只有 EgoSchema、MVBench 和 Perception Test 显示出从增加视频数据中获得的一致收益，我们假设这一现象与训练视频的底层视频分布有关。

### 为 \vsisuper Recall 设计的记忆框架
\label{appx:pred_sensing:vso_mem_design}

正如正文所述（并如 Algorithm~\ref{algo:pred_sensing:vso_mem_design} 所示），我们的预测记忆机制包含三个不同的记忆层级（$M_s$, $M_l$, $M_w$）和四个控制它们交互的关键转换函数：*感官流 (Sensory Streaming)*、*记忆压缩 (Memory Compression)*、*记忆巩固 (Memory Consolidation)* 以及 *检索 (Retrieval)*。本节将详细介绍这些函数的实现。

\paragraph{基本记忆单元。}
在我们的实现中，我们将来自每个大语言模型 (LLM) 层的*编码后的键值对 (key-value pairs)* 作为基本记忆单元。选择这种方式，而不是使用视觉编码器或视觉-语言连接器的输出潜特征，使我们能够充分利用 LLM 构建记忆的内部能力，而无需外部模块。这一设计决策将在后续章节中详细阐述。

\paragraph{流式感知。}
每个输入的帧最初由视觉编码器和视觉-语言连接器以 $W_s$ 的窗口大小独立处理。随后，它由 LLM 进一步编码，并参考选定的前序帧。
这些前序帧的键值对缓存于*感官记忆缓冲区* ($M_s$) 中，为该编码步骤提供必要的上下文。

\paragraph{基于惊讶度的记忆压缩。}
在对单帧进行编码的同时，我们会评估其“惊讶度”水平。
这是通过计算模型对当前帧的预测与实际地面真值观测之间的差异（均在潜特征空间中）来实现的。
当时间戳为 $t$ 的帧从感官记忆缓冲区 $M_s$ 移动到长期记忆 $M_l$ 时，如果它被认为是“非惊讶”的（即其惊讶度分数低于预定义的阈值 $T_s$），我们将沿空间 ($H\times W$) 维度对其键值对进行 2 倍下采样。
这种基于惊讶度的压缩减轻了存储在 $M_l$ 中信息的冗余。

\paragraph{基于惊讶度的记忆巩固。}
长期记忆 $M_l$ 初始化时带有一个预定义的预算大小 $B_{long}$（例如 32,768 个 token）。
当记忆 token 的体积超过此预算时，我们对 $M_l$ 应用一个*基于惊讶度*的巩固函数，以确保其保持在分配的限制内。
我们的巩固函数简单而有效：我们识别 $M_l$ 中每帧相关的惊讶度分数。
然后，移除（或“遗忘”）惊讶度分数最低的帧。
接着，我们根据这些帧的惊讶度分数进行合并或丢弃（我们在这里尝试了三种不同的策略：1. 遗忘最早的记忆，2. 遗忘惊讶度最低的记忆，以及 3. 遗忘惊讶度最低的记忆，同时如果存在相邻的惊讶记忆则将其合并）。
此过程循环往复，直到 $M_l$ 的总大小降至预算以下。

\paragraph{检索。}
在收到用户查询 $q$ 时，我们首先从长期记忆 ($M_l$) 中检索最相关的帧以构建工作记忆 ($M_w$)。
该 $M_w$ 随后作为回答用户查询的上下文。
为了在不借助外部模块的情况下高效执行此类检索，我们利用了 LLM 注意力机制固有的相似性度量能力。
具体而言，对于每个 Transformer 层，用户查询 $q$ 被转换为注意力机制的查询特征空间。
然后，我们计算该查询特征与存储在 $M_l$ 中的每帧的键特征之间的相似度。
相似度使用余弦距离进行衡量，为简单起见，多头特征被视为单个特征。
相似度分数最高的 $k$ 帧的键值对将被选中，并被注意力机制用于进一步编码用户查询。

\begin{algorithm}[H]
    \small
    \caption{**为 \vsisuper Recall 设计的记忆框架。**}
    \label{algo:pred_sensing:vso_mem_design}
    \KwIn{视频帧 $\{f_1, \dots, f_T\}$, 用户查询 $q$}
    \KwIn{编码器 $\mathcal{E}$, 解码器 $\mathcal{D}$, 惊讶度估计器 $\mathcal{S}$, 惊讶度阈值 $\tau$}
    \KwIn{压缩函数 $\mathcal{C}$, 巩固函数 $\mathcal{G}$, 检索函数 $\mathcal{R}$}
    \KwIn{感官记忆 $\mathcal{M}_s \gets \emptyset$ (预算为 $B_s$), 长期记忆 $\mathcal{M}_l \gets \emptyset$ (预算为 $B_l$), 工作记忆 $\mathcal{M}_w \gets \emptyset$}
    
    \For{$t \gets 1$ \KwTo $T$}{
        $z_t \gets \mathcal{E}(f_t, \mathcal{M}_s)$\;
        $\mathcal{M}_s \gets \mathcal{M}_s \cup \{z_t\}$ \tcp*[r]{流式感知}
        
        $s_t \gets \mathcal{S}(f_t, \mathcal{M}_s)$ \tcp*[r]{惊讶度估计}
        
        \While{$|\mathcal{M}_s| > B_s$}{
            从 $\mathcal{M}_s$ 中弹出最早的 $z_{\text{old}}$\;
            
            $m \gets \mathbf{1}[s_t \geq \tau] \cdot z_{\text{old}} + \mathbf{1}[s_t < \tau] \cdot \mathcal{C}(z_{\text{old}})$ \tcp*[r]{选择性压缩}
            
            $\mathcal{M}_l \gets \mathcal{M}_l \cup \{m\}$\;
            
            \If{$|\mathcal{M}_l| > B_l$}{
                $\mathcal{M}_l \gets \mathcal{G}(\mathcal{M}_l)$ \tcp*[r]{记忆巩固}
            }
        }
    }
    
    $\mathcal{M}_w \gets \mathcal{R}(q, \mathcal{M}_l)$ \tcp*[r]{检索工作记忆}
    $\hat{a} \gets \mathcal{D}(q, \mathcal{M}_w)$ \tcp*[r]{利用 $\mathcal{M}_w$ 回答查询}
    \Return $\hat{a}$
\end{algorithm}

### 为 \vsisuper Count 设计的智能体框架
\label{appx:pred_sensing:vsc_agentic_design}

Algorithm~\ref{algo:pred_sensing:vsc_agentic_design} 展示了我们为 \vsisuper Count 任务设计的智能体（agentic）框架。类似于 Algorithm~\ref{algo:pred_sensing:vso_mem_design} 中的记忆设计，我们使用滑动窗口方法（窗口大小为 $W_s$）对感官帧进行编码。
潜空间帧预测模块不断估计预期的下一帧，并计算预测误差以量化实际下一帧的“惊讶”程度。随着新帧的到达，超出感官记忆窗口的最早帧将被弹出并存储在长期记忆中。
如果弹出的帧被认为是“惊讶”的（即其预测误差超过预定义阈值 $\tau$），这可能表明存在场景或空间边界，我们会利用累积的长期记忆触发查询响应，并在响应后将其重置。生成的响应随后存储在答案记忆库中。最终答案计算为存储在该库中所有中间答案的总和。

\begin{algorithm}[H]
    \small
    \caption{**为 \vsisuper Count 任务设计的智能体框架。**}
    \label{algo:pred_sensing:vsc_agentic_design}
    \KwIn{视频帧 $\{f_1, \dots, f_T\}$, 用户查询 $q$}
    \KwIn{编码器 $\mathcal{E}$, 解码器 $\mathcal{D}$, 惊讶度估计器 $\mathcal{S}$, 阈值 $\tau$}
    \KwIn{感官记忆 $\mathcal{M}_s \gets \emptyset$ (预算为 $B_s$)}
    \KwIn{长期记忆 $\mathcal{M}_l \gets \emptyset$, 答案记忆库 $\mathcal{M}_{\text{Ans}} \gets \emptyset$}

    \For{$t \gets 1$ \KwTo $T$}{
        $z_t \gets \mathcal{E}(f_t, \mathcal{M}_s)$\;
        $\mathcal{M}_s \gets \mathcal{M}_s \cup \{z_t\}$ \tcp*[r]{流式感知}

        $s_t \gets \mathcal{S}(f_t, \mathcal{M}_s)$ \tcp*[r]{惊讶度估计}

        \If{$|\mathcal{M}_s| > B_s$}{
            从 $\mathcal{M}_s$ 中移除最早的 $z_{\text{old}}$\;
            $\mathcal{M}_l \gets \mathcal{M}_l \cup \{z_{\text{old}}\}$ \tcp*[r]{存储至长期记忆}
        }

        \If{$s_t \geq \tau$}{
            $\hat{a} \gets \mathcal{D}(q, \mathcal{M}_l)$ \tcp*[r]{使用长期记忆回答查询}
            $\mathcal{M}_{\text{Ans}} \gets \mathcal{M}_{\text{Ans}} \cup \{\hat{a}\}$\;
            $\mathcal{M}_l \gets \emptyset$ \tcp*[r]{重置长期记忆}
        }
    }

    \Return \texttt{Sum}($\mathcal{M}_{\text{Ans}}$)
\end{algorithm}
